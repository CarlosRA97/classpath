Classpath Project Status as of January 1, 1999

December was a big month for us. Aaron Renn finished up the integration of Classpath with the Japhar JVM. Patches have been submitted to the Japhar folks and will hopefully be applied to their CVS tree soon. With this patch applied, you can actually use Classpath as your Java library for Japhar.

Substantial progress was also made on the Classpath build process. Brian Jones cleaned up the autoconf/automake items and used the JavaDeps tool to generate correct dependency information for the build. We are now capable of creating a standard "tar.gz" distribution file that compiles out of the box like any other GNU package (ie, "configure; make; make install")

Jochen Hoenicke contributed a large chunk of java.util classes that round out that package for us.

What all this means is that a first development/alpha release of Classpath that works with Japhar will be available extremely soon. Probably as soon as the necessary patches are accepted into Japhar. This release will include the following:

  • java.beans
  • java.io
  • java.lang
  • java.lang.reflect
  • java.math
  • java.net
  • java.security (partial)
  • java.util
  • java.util.zip (maybe)

In addition to this first release, Paul Fisher and Jim Blair are making excellent progress on their set of GTK+ peers. Next up on their plate is the actual java.awt.* implementation.

In other big news, the Classpath project is going to be cooperating with Cygnus Solutions and Transvirtual Technologies in order to produce a unified free test suite for Java. This has developed into project Mauve. There isn't much at the website just yet, but there is a mailing list and some code!

Here is the status on a package by package basis. Note that there are still several package that aren't being worked on and so we still need volunteers.

java.applet

Greg Smith has a start at a java.applet package and is working at writing an appletviewer program for Classpath.

java.awt

Jim Blair and Paul Fisher continue to work on the GTK peers. The current work is progessing nicely.

java.beans

John Keiser has completed this package.

java.io

Full Java 1.2 implementation complete.

java.lang

Mostly Java 1.2 implementation complete. A few still do holes remain.

java.lang.reflect

Full Java 1.2 implementation complete

java.math

Paul Fisher completed a version of this class using the GNU MP library.

java.net

Mostly Java 1.2 implementation complete. Missing JarURLConnection and URLClassLoader. Included are protocol handlers for the "http" and "file" protocol, and a "text/plain" content handler. More need to be written, however. Note also that the HTTP protocol handler only supports the "GET" request method.

java.security.*

Aaron Renn is started work on a Java 1.2 version of this. The interface only packages (java.security.acl and java.security.interfaces) are complete. Work has been suspended here in order to concentrate on classes relevant to the first development release.

java.sql

James Cooper is working on this.

java.text

Aaron has written all of the collation classes in this package and is planning on finishing off the formatting classes that Wes Biggs started. These are totally untested, however.

java.util

Lots of people have contributed to this package, which is now almost a full Java 1.2 implementation.

java.util.zip

Loren Peace is almost done with this class.
