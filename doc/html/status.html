    • Classpath Project Status as of September 1, 1998

    We continue to make progress working on the core classes. The major issues facing the team are how to integrate with the JVM for those classes (such as java.lang.Runtime) that require it and how we will build the code. The JDK javac program chokes trying to compile key packages such as java.lang and java.io.

    Here is the status on a package by package basis. Note that there are still several package that aren't being worked on and so we still need volunteers.

    java.applet

    Greg Smith has a start at a java.applet package and is working at writing an appletviewer program for Classpath.

    java.awt

    Doug South is working on a set of GTK peers he calls GAWT.

    java.beans

    John Keiser has a working version of this that is complete. He is now working on some performance enhancements. He is also trying to clarify some ambiguities not fully spelled out in the spec.

    java.io - Serialization

    Geoff Berry is nearly complete with a Java 1.2 serialization implementation. Java 1.1 is already done. Serialization was a major worry of ours. Kudos to Geoff for knocking this one off.

    java.io

    A mostly Java 1.2 compatible version of this package is complete. It is only missing permission classes and a few new changes in JDK 1.2beta4. Aaron Renn plans on completing those once it becomes clear that Java 1.2 is stable.

    java.lang

    Paul Fisher and John Keiser continue to plug away at this. Many of the difficult classes such as Character and String are complete. The design of classes that need to interface to the JVM is still being debated.

    java.lang.reflect

    John Keiser has finished a version of this, but it is largely untested. He plans to complete the integration with Japhar and test RSN.

    java.math

    Paul Fisher completed a version of this class using the GNU MP library.

    java.net

    Aaron Renn has completed a mostly complete Java 1.2 version of this. It is only missing URLClassLoader and miscellaneous changes that are new with the JDK 1.2beta4. Also, this package needs many more protocol and content handlers. The only supported protocol currently is http and only the GET request method is supported there. Only text/plain is supported with a content handler.

    java.security.*

    Aaron Renn is currently working on a Java 1.2 version of this. The interface only packages (java.security.acl and java.security.interfaces) are complete.

    java.sql

    James Cooper is working on this

    java.text

    Wes Biggs recently started work on this class. Thanks to Wes for being brave enough to risk brain damaged by coding a package this poorly designed.

    java.util

    Stuart Ballard as written much of the collection classes. Greg Smith has also done some work here. Rumor has it there are also third party implementations we could use. A direction for how we finish off this package needs to be determined.
